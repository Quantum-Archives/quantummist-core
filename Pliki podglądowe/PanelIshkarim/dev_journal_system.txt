# 🪶 DEV_JOURNAL_SYSTEM.md – System dzienników użytkownika i dewelopera w QuantumMist

## 🎯 Cel
Opisać mechanikę działania dzienników:
- użytkownika (prywatne kanwy, notatki z czatów, refleksje),
- deweloperskich (logika systemu, zmiany, wersje).

---

## 🧍 Dziennik użytkownika (User Journal)

### 📁 Struktura:
```
/user_journals/
  └── {user_id}/
       ├── journal_meta.json
       ├── entry_001.md
       ├── entry_002.md
       └── ...
```

### ✨ Funkcje:
- Tworzenie wpisu na podstawie rozmowy (czat → wpis)
- Podział wpisu na sekcje (nagłówek, temat, refleksje, załączniki)
- Możliwość dodawania komentarzy do każdej sekcji
- Eksport wpisu jako PDF lub HTML

### 📦 Przyszłe opcje:
- Podgląd AI (sugestie, streszczenia, pytania pogłębiające)
- Współpraca z systemem kanw (export → /kanwy/)
- Tryb medytacyjny: dziennik w trybie offline z oprawą dźwiękową/obrazową

---

## 🧑‍💻 Dziennik deweloperski (Dev Log)

### 📁 Struktura:
```
/devlog/
  ├── CHAT_LOGIC.md
  ├── USER_PANEL_SLOTS.md
  ├── FILE_SYNC_LOGIC.md
  └── ...
```

### 🔁 Funkcje:
- Zapisywanie zmian w systemie (nowe pliki, API, decyzje)
- Historia refaktorów i błędów
- Połączenie z Ishkarim Terminalem (komenda `devlog write` tworzy nowy wpis)

### 📜 Format wpisu:
```md
## [DATA] – [TEMAT]

🔧 Co zmieniono: 

❓ Dlaczego: 

📌 Skutki: 

🧠 Komentarz: 
```

---

## 🔗 Sloty podpięcia:
- **W czacie**: `#dev_journal_captures`
- **W panelu użytkownika**: przycisk „Zapisz rozmowę do dziennika”
- **W terminalu**: `ishkarim devlog write --tag czat`
